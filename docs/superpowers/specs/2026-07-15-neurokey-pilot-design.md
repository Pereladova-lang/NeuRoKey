# NeuRoKey Пилот v1 — Дизайн-спецификация

**Дата:** 2026-07-15
**Статус:** Утверждено пользователем
**Область:** Пилотная веб-версия для детей 11–14 лет

## 1. Цель и объём

Веб-приложение-тренажёр для проверки главной гипотезы: интересны ли трёхслойные
нейрокогнитивные упражнения детям 11–14 лет и видят ли родители ценность в отчётах.

**Включено:** детский интерфейс (упражнения, прогресс, мотивация), минимальный
родительский портал (1 страница статистики), подписка через ЮKassa с 3-дневным триалом.

**Исключено (сознательно):** блокировка контента, ИИ-анализ, чат-бот (вместо него
статичный персонаж-маскот), мобильные приложения, другие возрастные когорты,
email-рассылки.

## 2. Упражнения (3 типа, трёхслойные)

| Тип | Академический слой | Когнитивный слой | Механика |
|---|---|---|---|
| **Комикс-детектив** (`comic`) | Смысловое чтение | Логика, прогнозирование | Комикс 4–6 панелей, часть реплик скрыта — выбрать верную из 3 вариантов |
| **Аналитик данных** (`data`) | Математика, статистика | Критическое мышление | График/таблица + вопросы «что можно заключить / что нельзя» |
| **Робо-логика** (`robot`) | Основы алгоритмов | Планирование, контроль импульсов | Составить последовательность команд для робота через поле с ловушками; энергия ограничена |

- Контент: рукописные JSON-пулы, 10 вариантов × 3 уровня сложности × 3 типа = 90 записей на старте.
- Сессия = 3 упражнения (~2 мин каждое), итого 5–7 минут. Жёсткий потолок по research.md.
- Никаких красных крестов при ошибке: подсказка + повторная попытка.
- Никаких лидербордов и сравнений с другими детьми.

## 3. Адаптивность (rule-based, без ИИ)

- Каждый результат пишет очки в 4 шкалы: внимание, память, логика, контроль.
- Ротация: тип вчерашнего первого упражнения не ставится первым сегодня.
- Сложность: точность >80% на уровне 3 раза подряд → уровень +1; <50% → уровень −1.
- Алгоритм прозрачен и объясним родителю (триггер доверия).

## 4. Мотивация

- Стрик (дни подряд с сессией), бейджи за серии, уровни маскота («Новичок» → «Мастер фокуса»).
- Маскот со статичными репликами (без LLM): приветствие, похвала, мягкая реакция на перерыв.
- Позитивное подкрепление, ноль наказаний.

## 5. Подписка (ЮKassa)

- **Trial:** 3 дня полного доступа без карты; старт = регистрация родителя.
  Длина триала — конфиг в БД (для A/B 1 vs 3 дня).
- **Тариф:** один, 299 ₽/месяц, рекуррентный.
- **Free tier после триала:** 1 упражнение в день.
- **Механика:** первый платёж с `save_payment_method: true`, далее автосписания по
  `payment_method_id`. Webhooks `payment.succeeded`/`payment.canceled` → статус в БД.
- **Статусы:** `trial` → `active` → `past_due` (3 дня грейс) → `expired`/`canceled`.
- Отмена в один клик в родительском портале.
- Ребёнку после окончания триала — мягкий экран «Попроси родителя продлить» (без вины).
- Fallback: фоновая сверка статусов платежей при заходе родителя в портал
  (на случай потерянного webhook).

## 6. Стек

- Next.js 15 (App Router) + TypeScript + Tailwind CSS + shadcn/ui
- SQLite через Prisma (переезд на Postgres = смена конфига)
- NextAuth: родитель — email+пароль; ребёнок — выбор профиля + PIN (4 цифры)
- ЮKassa через API-роуты Next.js
- Один репозиторий, один деплой (VPS или Vercel)

## 7. Модель данных

```
Parent        — id, email, passwordHash, createdAt
Child         — id, parentId, name, age, pin, mascotLevel, streak
Subscription  — id, parentId, status, trialEndsAt, paymentMethodId, nextBillingAt
Exercise      — id, type (comic/data/robot), level (1-3), contentJson
Session       — id, childId, startedAt, finishedAt, feedbackEmoji
SessionResult — id, sessionId, exerciseId, accuracy, durationSec,
                scoresJson ({attention, memory, logic, control})
Achievement   — id, childId, badgeKey, earnedAt
```

## 8. Экраны (9)

**Детские (6):**
1. Выбор профиля + PIN
2. Главный: маскот, «Начать сессию», стрик
3–5. Экраны трёх упражнений
6. Итоги сессии: что развивал (упрощённо) + эмодзи-фидбек (😍😊😐😕)
+ Карта прогресса/бейджи (доступна с главного)

**Родительские (3):**
1. Регистрация/вход
2. Дашборд: 4 шкалы с недельной динамикой, сессии за неделю, статус подписки
3. Подписка: оплата ЮKassa, отмена, история платежей

## 9. Обработка ошибок

- Прерванная сессия: сохраняется частично, продолжение доступно в течение часа.
- Потерянный webhook: сверка статусов при заходе в родительский портал.
- Ошибки оплаты: статус `past_due`, 3 дня грейс, затем `expired`.

## 10. Тестирование

- **Unit:** логика подбора сложности, начисление очков по шкалам (ядро системы).
- **Integration:** API сессий, жизненный цикл подписки (trial→active→expired).
- **E2E (Playwright):** happy-path одной детской сессии от входа до фидбека.

## 11. Дизайн

- Светлая тема, акцент #6366F1 (фиолетовый), секондари #06B6D4.
- Типографика Inter, крупные размеры для детского интерфейса (18px+).
- Микро-переходы 200–300ms, touch targets ≥44px, WCAG AA контраст.
