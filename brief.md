## <a name="обзор"></a>🎯 Обзор проекта

### Название
**NeuRoKey** — Нейроадаптивная образовательная среда

### Один лайнер
> "Телефон становится не источником цифровой зависимости, а персональным тренером мозга."

### Описание
NeuRoKey — это не родительский контроль, а первая цифровая нейроадаптивная образовательная платформа, которая управляет доступом к цифровому удовольствию (YouTube, TikTok, игры) через развитие мозга.

Ключевой принцип: **Перед доступом к высокодофаминовому контенту пользователь проходит короткую адаптивную нейрокогнитивную сессию**, которая одновременно:
- Развивает исполнительные функции мозга
- Закрепляет школьные знания
- Компенсирует последствия цифровой нагрузки
- Формирует здоровые привычки использования смартфона

### Целевая аудитория

| Сегмент | Характеристика |
|---------|-----------------|
| **Основная** | Дети 7–15 лет (школьники, зависимые от гаджетов) |
| **Родители** | 30–50 лет, образованные, озабочены экранной зависимостью |
| **B2B** | Психологи, нейропсycholog, школы (для реабилитации внимания) |

### Рыночный размер
- 🌍 **Глобально**: ~2 млрд школьников, 70% с доступом в интернет
- 🇷🇺 **Россия**: ~20 млн школьников, 85% с активным использованием соцсетей
- 📱 **TAM**: Родители готовы платить €5–10/месяц за решение

## <a name="функции"></a>⚙️ Функциональные требования

### 1. Ядро приложения: Система упражнений

#### 1.1 Трёхслойная архитектура упражнения

Каждое упражнение работает на трёх уровнях ОДНОВРЕМЕННО:

```
УПРАЖНЕНИЕ
├─ Уровень 1: Академический
│  └─ Закрепление школьной программы (математика, английский, чтение)
├─ Уровень 2: Когнитивный
│  └─ Развитие функций мозга (внимание, память, планирование)
└─ Уровень 3: Компенсаторный
   └─ Восстановление после цифровой нагрузки (YouTube, TikTok, игры)
```

**Требования:**
- [ ] Каждое упражнение имеет три связанных потока контента
- [ ] Ребёнок видит только игру (не видит архитектуру)
- [ ] Система отслеживает все три слоя параллельно
- [ ] Backend может анализировать эффективность каждого слоя

#### 1.2 Упражнения (примеры для MVP)

**Для возраста 7–10 лет:**

| Упражнение | Академический слой | Когнитивный слой | Компенсаторный слой |
|-----------|-------------------|------------------|-------------------|
| **Ловля рыб** | Таблица деления | Избирательное внимание | После YouTube (пассивная) |
| **Поиск лисы** | Английский (Find the fox) | Рабочая память + планирование | После YouTube (переключение) |
| **Робот на маршруте** | Логика пути | Планирование + контроль импульсов | После игр (высокая нагрузка) |

**Для возраста 11–14 лет:**

| Упражнение | Тип |
|-----------|-----|
| Комикс с пропусками | Понимание текста + логика |
| Анализ статистики | Критическое мышление + математика |
| Задача на кодирование | Логика + планирование |

#### 1.3 Адаптивность упражнения

**ИИ анализирует** экранную нагрузку → **подбирает** компенсирующее упражнение

```
Сценарий: YouTube 45 минут
├─ Система определяет: фиксация взгляда, частые переключения
├─ Вывод: нужно развивать "устойчивое внимание"
└─ Действие: рекомендует упражнение на длительную концентрацию

Сценарий: TikTok 1 час
├─ Система определяет: переключение внимания каждые 8 сек
├─ Вывод: нужно развивать "контроль импульсов"
└─ Действие: рекомендует упражнение с отвлекающими элементами
```

**Требования:**
- [ ] ИИ модель анализирует тип экранной нагрузки (YouTube, TikTok, игры, чтение)
- [ ] База знаний: какую нагрузку вызывает каждый контент
- [ ] Алгоритм подбора: какое упражнение компенсирует какую нагрузку
- [ ] Вариативность: не одно и то же упражнение каждый раз

**Требования:**
- [ ] Минимализм: максимум 2 кнопки на главном экране
- [ ] Видна информация: "Ты был в [контент] [время]"
- [ ] Одно рекомендуемое упражнение (алгоритм подбора)
- [ ] Для первой недели: возможность "Выбрать другое"
- [ ] После недели: блокировка контента (soft block с опцией)

#### 2.2 Экран упражнения

**Требования:**
- [ ] Полноэкранное игровое окружение (минимум отвлечений)
- [ ] Ясная инструкция (текст + иконка)
- [ ] Feedback по каждому действию:
  - ✅ Правильный ответ → "Отлично! Дальше!" (без баллов)
  - ❌ Неправильный → подсказка, повторная попытка (не штраф)
- [ ] Естественная точка выхода (упражнение заканчивается плавно)
- [ ] Micro-interactions: масштабирование кнопок, плавные переходы (200-300ms)

#### 2.3 Экран итогов (Post-Session)

```
┌──────────────────────────┐
│  Сессия завершена! ✅   │
│                          │
│  Сегодня ты:            │
│  • Развивал внимание     │

│                          │
│  [Открыть YouTube]      │

```

**Требования:**
- [ ] Показать, что произошло (упрощённый язык)
- [ ] Обратная связь от ребёнка (смайлики, не текст)
- [ ] Прямой доступ к контенту ("Открыть YouTube")
- [ ] Никаких лидербордов, никакого сравнения с другими

```
┌─────────────────────────────────────┐

├─────────────────────────────────────┤
│                                     │

│                                     │
│  НЕДЕЛЯ:                           │
│  Внимание:      ████████░  80% ↑18%│
│  Память:        ██████░░░  65% ↑12%│
│  Контроль:      █████░░░░░  52% ↑8%│
│                                     │
│  [Подробный отчёт]                │
│  [Настройки]                       │
└─────────────────────────────────────┘
```

**Требования:**
- [ ] Входной портал (отдельный от детского интерфейса)
- [ ] Видна нагрузка: какой контент, сколько времени
- [ ] Видны результаты: метрики когнитивных функций
- [ ] Еженедельный отчёт (не ежедневный!)
- [ ] Тренды: улучшение/ухудшение за неделю (%)
- [ ] Никаких баллов, только объективные метрики

#### 3.2 Настройки (Parent Settings)

**Требования:**
- [ ] Какой контент разблокируется после упражнения (YouTube, TikTok, Twitch, игры)
- [ ] Временные окна (когда доступен контент)
- [ ] Сложность упражнений (легко/среднее/сложно) — опционально
- [ ] Режим "Спокойный интерфейс" (меньше анимаций для ADHD)
- [ ] Экспорт данных (GDPR)
- [ ] Logout (безопасность)

#### 3.3 Еженедельный отчёт (Email)

**Требования:**
- [ ] Отправляется один раз в неделю (выбранный день)
- [ ] Содержит: прогресс, тренды, рекомендации
- [ ] Не пугает цифрами (максимум 3 метрики)
- [ ] Содержит пример: "Маша улучшила внимание на 18%"
- [ ] Опция отписки

### 4. Система аналитики и метрик

#### 4.1 Встроенные тесты когнитивных функций

**Раз в месяц (не чаще):**

| Тест | Измеряет | Длительность |
|------|----------|--------------|
| **Тест внимания** | Скорость обработки информации | 2 мин |
| **Тест памяти** | Объём рабочей памяти | 2 мин |
| **Тест планирования** | Способность к планированию действий | 2 мин |

**Требования:**
- [ ] Тесты не выглядят как "оценка" (это "проверка прогресса")
- [ ] Результаты сравниваются с базовой линией (месяц назад)
- [ ] Прогресс показывается в %, не в баллах
- [ ] Результаты видны в родительском портале

#### 4.2 Скрытая аналитика (backend)

**Что отслеживать:**
- [ ] Время сессии, точность, скорость выполнения
- [ ] Какие упражнения выбирает ребёнок (если есть выбор)
- [ ] Какие упражнения пропускаются (индикатор скуки)
- [ ] Feedback от ребёнка (смайлики после сессии)
- [ ] Время до следующего захода (engagement)
- [ ] Errors и bugs (техническая стабильность)

### 5. Система доступа и блокировки контента

#### 5.1 Блокировка контента (Soft Block)

**Требования:**
- [ ] После выбора контента (YouTube, TikTok): "Сначала сессия!" (мягко)
- [ ] Ребёнок может пропустить первые 7 дней (зацепка)
- [ ] После дня 7: система мягко предлагает сессию
- [ ] После дня 14: блокировка становится жёсткой (но есть override родителя)
- [ ] Блокировка НЕ выглядит как наказание (фреймировка: "Давай разогреем мозг!")

#### 5.2 Интеграция с парентальным контролем

**Требования:**
- [ ] Google Family Link (Android): синхронизация экранного времени
- [ ] iOS Screen Time: получение данных о нагрузке
- [ ] Логирование всех открытий контента
- [ ] API для передачи данных третьим лицам (школы, психологи)

### 3. Data Flow

```
СЦЕНАРИЙ: Ребёнок открывает YouTube

1. [FRONTEND] Ребёнок открывает YouTube в приложении NeuRoKey
2. [BACKEND] Запрос: "Дать упражнение для YouTube"
3. [ML SERVICE] ИИ анализирует:
   - Последнее использование YouTube (в истории)
   - Предыдущую нагрузку (5+ сессий)
   - Какие функции перегружены
4. [BACKEND] Выбирает рекомендуемое упражнение
5. [FRONTEND] Показывает упражнение ребёнку
6. [DURING EXERCISE] Отслеживает:
   - Точность ответов
   - Время выполнения
   - Ошибки и попытки
7. [AFTER EXERCISE] Записывает в БД:
   - Результаты по всем 3 слоям
   - Feedback от ребёнка (смайлик)
   - Timestamp и duration
8. [PARENT PORTAL] Обновляется метрика в родительском отчёте
```

## <a name="mvp"></a>🚀 MVP Scope (Минимально жизнеспособный продукт)

### MVP = Месяц разработки (4 недели)

#### Что ВКЛЮЧЕНО в MVP:

**Возрастные когорты:**
- [ ] 7–10 лет (основной фокус)
- [ ] 11–14 лет (вторичный)

**Упражнения (3 для каждой когорты = 6 всего):**

| Возраст | Упражнение 1 | Упражнение 2 | Упражнение 3 |
|---------|-------------|-------------|-------------|
| 7–10 | Ловля рыб (мат) | Поиск животного (англ) | Робот на маршруте (логика) |
| 11–14 | Анализ комикса | Задача на кодирование | Интерпретация данных |

**Платформы:**
- [ ] iOS (основной фокус)
- [ ] Android (вторичный)
- [ ] Web (базовый интерфейс)

**Функции:**
- [ ] Детский интерфейс (упражнения)
- [ ] Родительский портал (базовый)
- [ ] Система учёта упражнений (БД)
- [ ] Неделю бесплатный доступ
- [ ] Email отчёты (еженедельные)
- [ ] Google/Apple auth (облегчённо)

**NOT включено в MVP:**
- ❌ Полная ИИ адаптивность (упрощённая логика)
- ❌ Интеграция Family Link (ручной ввод)
- ❌ Встроенные тесты когнитивных функций
- ❌ B2B версия
- ❌ Множество упражнений (ротация вручную)

#### Timeline MVP:

```
Неделя 1: Архитектура + Setup
├─ Design system (цвета, типография, spacing)
├─ Проект структура (Next.js setup)
├─ Database schema
└─ Authentication
```

## <a name="риски"></a>⚠️ Риски и ограничения

### Критичные риски

| Риск | Вероятность | Воздействие | Митигация |
|------|-----------|-----------|-----------|
| **Родители видят это как ещё один контроль** | 40% | 🔴 High | Маркетинг: фокус на "тренер", не контроль |
| **Ребёнок находит способ обойти блокировку** | 30% | 🔴 High | Security audit, постоянные обновления |
| **Нет научных доказательств эффективности** | 25% | 🔴 High | Пилот с метриками, партнёрство с учёными |
| **Платежи не работают (Apple/Google Issues)** | 15% | 🟡 Medium | Тестирование, support из коробки |
| **Ребёнок отказывается заниматься** | 35% | 🟡 Medium | Ротация упражнений, gamification |

### Технические риски

| Риск | Вероятность | Решение |
|------|-----------|---------|
| **Масштабируемость (100k+ users)** | 20% | Kubernetes, Load balancing |
| **Базовые данные персонализированы неправильно** | 15% | Rigorous testing, validation |
| **Integration с Family Link fails** | 10% | Fallback (manual input) |
| **GDPR violations** | 5% | Legal review, compliance audit |

### Бизнес-риски

| Риск | Вероятность | Решение |
|------|-----------|---------|
| **Конкуренция от больших игроков** | 40% | Фокус на нише, научность |
| **Пользователь не готов платить** | 30% | Freemium model, 30-day guarantee |
| **Negative press (гаджеты плохие)** | 20% | Прозрачность, научный подход |
| **Недостаток финансирования** | 25% | Seed funding, grants |

### Операционные ограничения

| Ограничение | Статус | План |
|-----------|--------|------|
| **Команда малая (4-5 человек)** | ⚠️ | Нанять разработчиков + freelancers |
| **Первый продукт** | ⚠️ | Много обучения, требует patience |
| **Нет научного совета** | ⚠️ | Партнёрство с нейропсycholog |
| **Budget ограничен** | ⚠️ | Фокус на MVP, дешёвые облачные сервисы |
